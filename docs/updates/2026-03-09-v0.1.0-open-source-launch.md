# AgentOS v0.1.0 Open-Source Launch

- Date: 2026-03-09
- Release: `v0.1.0`
- Audience: developers, platform teams, early adopters

## Summary

AgentOS v0.1.0 is the first public release of the open-source core.

This release is intentionally positioned as a **community infrastructure baseline**, not a fully finished end-user product. The goal is to make the execution substrate public: control plane, worker runtime, scheduling, audit, telemetry, and the extension surfaces around them.

## What Is Included

The community core now includes:

- Go control plane with `apiserver`, `controller`, and CLI entry points
- Rust worker runtime for command execution
- local and NATS-backed scheduling paths
- task submission, planning, execution, replay, and audit APIs
- SSE telemetry for task and action streams
- agent loop / tool-calling support
- repository-level licensing, security, and contribution guidance

## What This Release Is For

This release is for teams that want to:

- self-host an agent execution substrate
- inspect how execution, scheduling, and audit are wired together
- experiment with governed agent workloads instead of ad-hoc scripts
- build extensions, adapters, or vertical workflows on top of the core

## What This Release Is Not

This release is not yet:

- a polished enterprise console
- a hosted cloud product
- a complete org-governance platform
- a packaged business workflow solution

Those directions may exist later, but the public repository is currently focused on the **community core**.

## Verification

The release was verified with:

- `go test ./... -count=1`
- `./scripts/acceptance.sh`

## Launch Positioning

Public messaging for `v0.1.0` should stay simple:

> AgentOS is an open-source agent execution platform with a Go control plane and a Rust runtime plane.

That framing keeps the project honest, technically credible, and consistent with the current repository boundary.
