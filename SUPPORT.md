# Support

## Scope

AgentOS community support covers the open-source repository core published under `Apache-2.0`.

This repository is best suited for:

- self-hosted evaluation of the control plane and worker runtime
- local and NATS-backed task scheduling
- task execution, audit, replay, and telemetry APIs
- contributions, bug reports, and feature proposals for the community core

## How To Get Help

### Bug Reports

Use the GitHub bug report template when:

- a documented flow does not work
- a test, command, or endpoint behaves incorrectly
- a regression appears in the repository core

Please include:

- the exact command or API request
- logs or error output
- environment details
- reproduction steps

### Feature Requests

Use the GitHub feature request template when:

- you want a new capability in the community core
- you want to improve developer experience, docs, or extension points
- you want to propose platform-level capabilities that fit the open-source boundary

### Security Issues

Do **not** open a public issue for vulnerabilities.

See `SECURITY.md` for the responsible disclosure process.

## What Community Support Does Not Include

Community support does not include:

- guaranteed response times or SLA
- private onboarding
- custom deployment work
- enterprise governance features
- hosted cloud operations

## Before Opening An Issue

Please check the following first:

- `README.md`
- `README_CN.md`
- `CONTRIBUTING.md`
- `docs/architecture/`
- existing GitHub issues and pull requests

## Commercial / Enterprise

Enterprise add-ons and hosted cloud offerings are outside the repository-core support boundary.

This repository currently provides community support for the open-source core only.
